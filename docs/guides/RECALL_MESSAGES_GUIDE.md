# 撤回消息功能使用说明

## 功能说明

`recall_messages.py` 脚本用于撤回飞书群聊中指定序号的消息。

## 权限要求

### 必需的飞书权限

要撤回群聊中的消息，需要满足以下**任一**条件：

#### 方案 1：使用用户身份（推荐）

**前提条件：**
- 授权用户必须是目标群聊的**管理员**或**群主**

**需要的权限：**
- ✅ `im:message` - 获取与发送消息（用户身份权限）
- ✅ `im:message:readonly` - 读取消息内容（用户身份权限）

**配置步骤：**
1. 在飞书开放平台 → 应用详情 → 权限管理
2. 确保已添加上述用户身份权限
3. 在飞书群聊中，确保授权用户是管理员
4. 重新运行授权：`python src/auth.py`

#### 方案 2：使用应用身份

**前提条件：**
- 应用机器人必须是目标群聊的**管理员**

**需要的权限：**
- ✅ `im:message` - 获取与发送消息（应用身份权限）

**配置步骤：**
1. 在飞书开放平台 → 应用详情 → 权限管理
2. 确保已添加上述应用身份权限
3. 在飞书群聊中，将机器人设置为管理员：
   - 打开群聊 → 群设置 → 群管理 → 添加管理员 → 选择你的应用机器人

### 重要限制

⚠️ **飞书 API 限制：**
- 只能撤回**自己发送**的消息
- 或者需要是**群管理员**才能撤回群内其他人的消息
- 无法撤回系统消息

如果撤回失败，可能的原因：
1. 授权用户不是群管理员
2. 应用机器人不是群管理员
3. 消息已被撤回
4. 权限不足

## 使用方法

### 1. 列出所有消息（查看序号）

首先查看所有消息及其序号：

```bash
python src/archive/recall_messages.py --list
```

输出示例：
```
============================================================
消息列表（共 50 条）
============================================================

[消息 1] 时间: 2024-02-09 10:00:00
  内容: 这是一条测试消息 https://mp.weixin.qq.com/s/abc123

[消息 2] 时间: 2024-02-09 10:15:30
  内容: 大家好，今天分享一些学习资料

[消息 3] 时间: 2024-02-09 10:30:45
  内容: 感谢分享！

...
```

### 2. 试运行模式（推荐先使用）

查看将要撤回哪些消息，但不实际撤回：

```bash
# 撤回单条消息
python src/archive/recall_messages.py --indices 1 --dry-run

# 撤回多条消息
python src/archive/recall_messages.py --indices 1,3,5 --dry-run

# 撤回范围内的消息
python src/archive/recall_messages.py --indices 1-5 --dry-run

# 混合使用
python src/archive/recall_messages.py --indices 1,3-5,7,10-15 --dry-run
```

### 3. 实际撤回

确认无误后，去掉 `--dry-run` 参数执行撤回：

```bash
# 撤回单条消息
python src/archive/recall_messages.py --indices 1

# 撤回多条消息
python src/archive/recall_messages.py --indices 1,3,5

# 撤回范围内的消息
python src/archive/recall_messages.py --indices 1-5

# 混合使用
python src/archive/recall_messages.py --indices 1,3-5,7,10-15
```

程序会要求输入 `yes` 确认撤回。

### 4. 逐条确认模式（推荐用于重要操作）

启用逐条确认模式，撤回每条消息前都会显示内容并要求确认：

```bash
python src/archive/recall_messages.py --indices 1-10 --confirm-each
```

**逐条确认的选项：**
- 输入 `y` - 撤回此消息
- 输入 `n` - 跳过此消息
- 输入 `q` - 退出撤回操作

## 安全特性

### 1. 倒序撤回

程序会自动从编号最大的消息开始撤回（倒序），避免撤回前面的消息后导致后面消息序号混乱。

**示例：**
```
要撤回的消息序号: [1, 3, 5, 7, 9]
实际撤回顺序: 9 → 7 → 5 → 3 → 1
```

### 2. 逐条确认

使用 `--confirm-each` 参数可以在撤回每条消息前单独确认：

```bash
python src/archive/recall_messages.py --indices 1,3,5 --confirm-each
```

**输出示例：**
```
[1/3] 消息 5
  内容: 这是第5条消息的内容...
  确认撤回此消息？(y=撤回, n=跳过, q=退出): y
  ✓ 撤回成功

[2/3] 消息 3
  内容: 这是第3条消息的内容...
  确认撤回此消息？(y=撤回, n=跳过, q=退出): n
  ⊘ 已跳过

[3/3] 消息 1
  内容: 这是第1条消息的内容...
  确认撤回此消息？(y=撤回, n=跳过, q=退出): q
  ✗ 用户选择退出，停止撤回
```

## 命令行参数

| 参数 | 说明 | 示例 |
|------|------|------|
| `--list` | 列出所有消息及其序号，不执行撤回 | `--list` |
| `--indices` | 指定要撤回的消息序号 | `--indices 1,3,5` |
| `--dry-run` | 试运行模式，只显示将要撤回的消息 | `--dry-run` |
| `--confirm-each` | 逐条确认模式，撤回每条消息前都需要确认 | `--confirm-each` |

### 消息序号格式

支持以下格式：

| 格式 | 说明 | 示例 |
|------|------|------|
| 单个序号 | 撤回一条消息 | `--indices 1` |
| 逗号分隔 | 撤回多条消息 | `--indices 1,3,5` |
| 范围 | 撤回连续的消息 | `--indices 1-5` |
| 混合 | 组合使用 | `--indices 1,3-5,7,10-15` |

## 工作流程

1. **获取群聊消息**：从目标群聊获取所有消息
2. **分配序号**：为每条消息分配序号（从 1 开始）
3. **列出消息**（可选）：使用 `--list` 查看所有消息
4. **筛选消息**：根据 `--indices` 参数筛选要撤回的消息
5. **倒序排列**：自动将消息按序号倒序排列（从大到小）
6. **显示列表**：展示将要撤回的消息详情
7. **确认撤回**：
   - 普通模式：一次性确认所有消息
   - 逐条确认模式（`--confirm-each`）：每条消息单独确认
8. **执行撤回**：按倒序逐条撤回消息并显示结果

**注意：** 撤回消息功能始终获取所有历史消息，不受主程序的 `last_processed_time` 限制。

## 安全提示

⚠️ **警告：撤回操作不可恢复！**

建议操作流程：
1. 先使用 `--list` 查看所有消息及其序号
2. 使用 `--dry-run` 查看将要撤回的消息
3. 确认无误后，再执行实际撤回
4. 建议先在测试群聊中测试

## 常见问题

### Q: 如何知道要撤回哪些消息的序号？

A: 先运行 `python src/archive/recall_messages.py --list` 查看所有消息及其序号。

### Q: 撤回失败提示 "no permission"？

A: 这表示权限不足。请确保：
1. 授权用户是群管理员（使用用户身份时）
2. 或应用机器人是群管理员（使用应用身份时）

### Q: 如何将机器人设置为群管理员？

A:
1. 打开飞书群聊
2. 点击右上角 → 群设置
3. 群管理 → 添加管理员
4. 选择你的应用机器人

### Q: 可以撤销撤回操作吗？

A: 不可以。撤回操作是永久性的，无法恢复。建议先使用 `--dry-run` 模式查看。

### Q: 为什么要倒序撤回消息？

A: 从编号最大的消息开始撤回，可以避免撤回前面的消息后，后面消息的实际位置发生变化导致误删。这是一个安全保护机制。

### Q: 什么时候应该使用 --confirm-each？

A: 建议在以下情况使用逐条确认模式：
- 撤回大量消息时
- 不确定要撤回哪些消息时
- 需要根据消息内容决定是否撤回时
- 进行重要操作时

### Q: 逐条确认时可以中途退出吗？

A: 可以。在确认提示时输入 `quit` 即可退出，已撤回的消息无法恢复，但未处理的消息会保留。

### Q: 消息序号是固定的吗？

A: 不是。消息序号只在本次运行中有效，每次运行都会重新分配序号。

### Q: 如何撤回所有消息？

A:
```bash
# 1. 查看消息总数
python src/archive/recall_messages.py --list

# 2. 假设有 50 条消息，撤回全部
python src/archive/recall_messages.py --indices 1-50 --dry-run
python src/archive/recall_messages.py --indices 1-50
```

### Q: 撤回速度慢吗？

A: 程序会逐条撤回消息，避免触发 API 限流。如果消息数量多，可能需要一些时间。

## 技术细节

### API 接口

使用飞书开放平台的消息撤回接口：

```
DELETE /open-apis/im/v1/messages/:message_id
```

### 权限检查

程序使用用户身份（user_access_token）进行撤回操作，需要：
- 用户必须有权限撤回该消息
- 通常需要是群管理员或消息发送者

### 错误处理

- 网络错误：自动重试
- 权限错误：记录并继续处理下一条
- Token 过期：自动刷新
