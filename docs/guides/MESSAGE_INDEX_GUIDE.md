# 消息序号功能说明

## 功能概述

程序现在会为每条消息分配一个序号，并在处理过程中显示未含链接的消息清单，同时在输出表格中记录每个链接对应的消息序号。

## 新增功能

### 1. 消息序号分配

- 所有获取到的消息按时间顺序从 1 开始编号
- 序号在整个处理过程中保持一致
- 同一条消息中的多个链接会有相同的消息序号

### 2. 未含链接的消息清单

在解析 URL 之前，程序会列出所有不包含链接的消息：

```
============================================================
未含链接的消息清单（共 15 条）
============================================================

[消息 2] 时间: 2024-02-09 10:15:30
  内容: 大家好，今天分享一些学习资料

[消息 5] 时间: 2024-02-09 10:30:45
  内容: 感谢分享！

[消息 8] 时间: 2024-02-09 11:00:20
  内容: 这个话题很有意思

...

============================================================
```

**显示内容：**
- 消息序号
- 发送时间
- 消息内容（超过 100 字符会截断）

### 3. 表格中的消息序号列

在飞书表格中新增"消息序号"列，记录每个链接来自哪条消息。

**表格列顺序：**
1. 消息序号
2. 文章标题
3. 文章发表日期
4. 星期
5. 文章链接URL
6. 摘录异常信息
7. 文章来源
8. 备注

## 使用场景

### 场景 1：追溯链接来源

当你在表格中看到某个链接时，可以通过"消息序号"快速找到它来自群聊中的哪条消息。

**示例：**
```
消息序号: 12
文章标题: 如何使用飞书 API
文章链接: https://mp.weixin.qq.com/s/abc123
```

这表示该链接来自第 12 条消息。

### 场景 2：识别批量分享

如果多个链接有相同的消息序号，说明它们来自同一条消息（批量分享）。

**示例：**
```
消息序号: 25 | 文章标题: Python 教程 | 链接: https://...
消息序号: 25 | 文章标题: Java 教程 | 链接: https://...
消息序号: 25 | 文章标题: Go 教程 | 链接: https://...
```

这 3 个链接都来自第 25 条消息。

### 场景 3：查看非链接消息

通过"未含链接的消息清单"，你可以：
- 了解群聊中有哪些非链接消息
- 检查是否有遗漏的重要信息
- 统计链接消息和非链接消息的比例

## 配置说明

### config.yaml 配置

已自动添加"消息序号"列配置：

```yaml
table_columns:
  message_index: 消息序号
  title: 文章标题
  publish_date: 文章发表日期
  weekday: 星期
  url: 文章链接URL
  error_info: 摘录异常信息
  source: 文章来源
  remark: 备注
```

### 表格表头要求

确保飞书表格的第一行包含"消息序号"列。如果没有，请手动添加。

**推荐表头顺序：**
```
消息序号 | 文章标题 | 文章发表日期 | 星期 | 文章链接URL | 摘录异常信息 | 文章来源 | 备注
```

## 注意事项

1. **消息序号是相对的**
   - 序号只在本次运行中有效
   - 下次运行会重新从 1 开始编号
   - 如果使用 `--all` 参数，序号会包含所有历史消息

2. **表格列顺序**
   - 程序会自动匹配表头中的列名
   - 列的顺序可以任意调整
   - 只要列名匹配 config.yaml 中的配置即可

3. **未含链接的消息**
   - 这些消息不会写入表格
   - 只在控制台显示
   - 可以用于了解群聊的整体情况

4. **消息内容截断**
   - 超过 100 字符的消息内容会被截断
   - 只影响控制台显示，不影响实际处理

## 常见问题

### Q: 为什么需要消息序号？

A: 消息序号可以帮助你：
- 追溯链接的来源
- 识别批量分享的消息
- 在群聊中快速定位原始消息

### Q: 消息序号会保存到配置文件吗？

A: 不会。消息序号只在本次运行中有效，不会持久化保存。

### Q: 如何在群聊中找到对应序号的消息？

A:
1. 记下消息的发送时间（在"未含链接的消息清单"中显示）
2. 在飞书群聊中按时间搜索
3. 或者根据消息内容搜索

### Q: 同一条消息中有多个链接怎么办？

A: 每个链接都会单独写入表格的一行，但它们的消息序号相同。

### Q: 表格中没有"消息序号"列怎么办？

A:
1. 手动在表格第一行添加"消息序号"列
2. 或者程序会自动跳过该列（如果 config.yaml 中配置了但表格中没有）
